# US-025: MKXL map interactions у combo builder

## Мета

Дати користувачу створювати, дублювати або ремонтувати `MKXL` combo з урахуванням конкретної карти, stage zone/segment і environment interactables.

## Архітектурний контекст

MKXL map interactions є частиною MKXL business scope:

- `mkxl/data` зберігає stage, zone, segment, interactable і stage-specific graph data;
- `mkxl/rules` визначає usage policy, spacing/state/frame constraints і validation rules;
- `mkxl/builder` компонує MKXL graph і повертає valid next moves;
- `@mk-combos/ui` лише рендерить whiteboard, internal move picker і frame meter.

Shared builder presentation не вирішує, які interactables доступні.

## Контекст

У `MKXL` частина routes залежить від карти. Кожна карта має власну `MKXLStageInteractionSchema`, яка описує zones, segments, interactables, stage-specific moves, frame-aware transitions і usage policy.

Stage context є optional:

- без selected stage MKXL builder працює тільки з character + variation graph;
- із selected stage `mkxl/builder` додає interaction schema цієї карти;
- selected zone, segment, distance band, corner side і facing визначають, які interactables доступні як valid next moves.

## Основний сценарій

1. Користувач відкриває `/mkxl/builder`.
2. App Shell резолвить `mkxlBusiness`.
3. Користувач обирає MKXL character і variation.
4. Користувач optional обирає stage, zone і segment.
5. Користувач задає runtime start state: meter, damage estimate, position, stance, player state, opponent state і frame context.
6. `mkxl/builder` компонує graph: base character graph, variation overlay і optional stage interaction schema.
7. `@mk-combos/ui` рендерить valid candidates, selected path, runtime summary і used interactables.
8. Якщо stage не вибрана, interactable moves не показуються.
9. Якщо stage вибрана, `mkxl/builder` повертає тільки interactables цієї карти, zone, segment і distance band.
10. Користувач додає moves через frame-aware valid transitions.
11. `mkxl/builder` оновлює `movePath`, `cachedNotation`, meter, damage estimate, position, stance, player/opponent state, frame context і used interactables.
12. Після finish `apps/web` зберігає custom combo у `games["mkxl"]`.

## Duplicate Flow

1. Користувач дублює stage-specific seeded combo з Catalog або Detail.
2. Builder відкривається з prefilled character, variation, `stageContext`, runtime summary і initial `movePath`.
3. `mkxl/builder` replay-ить initial path через актуальний composed graph.
4. Якщо path валідний, користувач може продовжити редагування або finish.
5. Якщо path stale, `@mk-combos/ui` показує invalid reason, original path, valid prefix і invalid boundary без видалення source combo.

## Винятки

- Якщо selected stage не має потрібного interactable, move не доступний у builder.
- Якщо interactable має usage policy `oncePerCombo` і вже використаний, `mkxl/builder` не пропонує його повторно.
- Якщо edge не проходить `frameWindow`, spacing constraints або state requirements, він не входить у valid next moves.
- Якщо користувач змінює character, variation, stage або start runtime state, `mkxl/builder` replay-ить current path і позначає stale/invalid state без автоматичного видалення combo.

## Критерії приймання

- Builder без selected stage не показує MKXL interactables.
- Builder із selected stage показує тільки interactables цієї карти.
- Zone, segment, distance band, corner side і facing впливають на доступність interactables.
- Frame-aware edge без valid `frameWindow` не доступний як next move.
- Duplicate stage-specific combo prefill-ить `stageContext`.
- Stale stage-specific path не видаляється і має readable invalid reason.
- Stale stage-specific path показує original path, valid prefix і invalid boundary у `UI-CMP-035 Combo Whiteboard`.
