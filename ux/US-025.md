# US-025: MKXL map interactions у combo builder

## Мета

Дати користувачу створювати, дублювати або ремонтувати `MKXL` combo з урахуванням конкретної карти, stage zone/segment і environment interactables.

## Контекст

У `MKXL` частина routes залежить від карти. Кожна карта має власну `MKXLStageInteractionSchema`, яка описує zones, segments, interactables, stage-specific moves, frame-aware transitions і usage policy.

Stage context є optional:

- без selected stage builder працює тільки з character + variation graph;
- із selected stage builder додає interaction schema цієї карти;
- selected zone, segment, distance band, corner side і facing визначають, які interactables доступні як valid next moves.

## Основний сценарій

1. Користувач відкриває `UI-PAGE-006 Custom Combo Builder`.
2. Користувач обирає `MKXL`, character і variation.
3. Користувач optional обирає stage, zone і segment.
4. Користувач задає runtime start state: meter, damage estimate, position, stance, player state, opponent state і frame context.
5. Builder компонує graph: base character graph, variation overlay і optional stage interaction schema.
6. `UI-CMP-024 Move Picker` показує only valid next moves.
7. `UI-CMP-035 Combo Whiteboard` показує selected path, runtime summary і used interactables.
8. Якщо stage не вибрана, interactable moves не показуються.
9. Якщо stage вибрана, builder показує тільки interactables цієї карти, zone, segment і distance band.
10. Користувач додає moves через frame-aware valid transitions.
11. Builder оновлює `movePath`, `cachedNotation`, meter, damage estimate, position, stance, player/opponent state, frame context і used interactables.
12. Після finish app зберігає custom combo з `movePath`, `cachedNotation`, `stageContext` і runtime summary.

## Duplicate flow

1. Користувач дублює stage-specific seeded combo з Catalog або Detail.
2. Builder відкривається з prefilled character, variation, `stageContext`, runtime summary і initial `movePath`.
3. Builder replay-ить initial path через актуальний composed graph.
4. Якщо path валідний, користувач може продовжити редагування або finish.
5. Якщо path stale, builder показує invalid reason, original path, valid prefix і invalid boundary у `UI-CMP-035 Combo Whiteboard` без видалення source combo.

## Винятки

- Якщо selected stage не має потрібного interactable, move не доступний у builder.
- Якщо interactable має usage policy `oncePerCombo` і вже використаний, builder не пропонує його повторно.
- Якщо edge не проходить `frameWindow`, spacing constraints або state requirements, він не входить у valid next moves.
- Якщо користувач змінює character, variation, stage або start runtime state, builder replay-ить current path і позначає stale/invalid state без автоматичного видалення combo.

## Критерії приймання

- Builder без selected stage не показує MKXL interactables.
- Builder із selected stage показує тільки interactables цієї карти.
- Zone, segment, distance band, corner side і facing впливають на доступність interactables.
- Frame-aware edge без valid `frameWindow` не доступний як next move.
- Duplicate stage-specific combo prefill-ить `stageContext`.
- Stale stage-specific path не видаляється і має readable invalid reason.
- Stale stage-specific path показує original path, valid prefix і invalid boundary у `UI-CMP-035 Combo Whiteboard`.
