# US-001: Перший запуск

## Метадані

- ID: `US-001`
- Категорія: `Контексти входу / Перший запуск`
- Статус деталізації: `Описано`
- Пов'язані сценарії: `US-002`, `US-003`, `US-008`, `US-009`, `US-021`, `US-023`

## Мета користувача

- Швидко зрозуміти, що застосунок є каталогом Mortal Kombat combo.
- Обрати базові налаштування перед першим входом у каталог.
- Перейти до вибору гри, персонажа і списку комбо після обов'язкового підтвердження.

## Передумови

- Користувач відкриває root URL застосунку.
- У браузері немає збережених settings або first-launch completion marker.
- Застосунок має доступ до seeded combo data.

## Основний flow

1. Користувач відкриває застосунок.
2. App визначає, що це перший запуск у цьому браузері.
3. Користувач бачить first-launch setup як обов'язковий стартовий стан.
4. Setup пропонує вибрати мову `EN/UA`, гру `MKXL/MK1` і display mode `FGC/PlayStation/Xbox`.
5. Користувач обов'язково підтверджує налаштування; пропуск сценарію недоступний.
6. App зберігає вибір у local browser settings.
7. Користувач переходить до каталогу обраної гри.
8. Далі користувач може вибрати персонажа й перейти до combo list.

## Альтернативні flow

- Користувач приймає preselected defaults: це дозволено, але все одно потрібне явне підтвердження.
- Користувач змінює мову або display mode після першого запуску через shell controls.
- Контролер уже підключений: app може показати controller hint, але це не блокує налаштування.
- `localStorage` недоступний: app дозволяє підтвердити session-only settings і повідомляє, що вибір не буде збережено.

## Out of scope

- Перший запуск за прямим посиланням.
- Повноцінний onboarding wizard.
- Імпорт backup.
- Створення custom combo.

## Acceptance criteria

- First-launch setup неможливо пропустити.
- Доступ до каталогу відкривається тільки після підтвердження language, game і display mode.
- Preselected defaults можуть бути підтверджені одним action, але не застосовуються як завершений first launch без явного підтвердження.
- Вибрані language, game і display mode застосовуються одразу.
- Після reload налаштування зберігаються, і setup не показується повторно.
- Користувач може перейти до `Game -> Character -> Combo list`.

## Test scenarios

- Fresh browser state відкриває root URL і бачить обов'язковий first-launch setup.
- Skip/close action відсутній або недоступний.
- Вибір `UA`, `MK1`, `Xbox` і підтвердження одразу змінюють інтерфейс і формат input display.
- Reload після підтвердження не показує first-launch setup повторно.
- Без підтвердження користувач не може перейти до catalog, character або combo detail.
- Root first launch не перехоплює deep link сценарії `US-023` і `US-024`.

## Assumptions

- Дефолтна мова визначається з browser locale, якщо підтримується `EN/UA`; fallback - `EN`.
- Дефолтний display mode - `FGC`.
- Дефолтна гра може бути preselected, але користувач все одно має підтвердити setup.
- First-launch setup є компактним mandatory gate, а не повноцінним onboarding wizard.
- Перший запуск визначається відсутністю local settings або first-launch completion marker.
