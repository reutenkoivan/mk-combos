# UX сценарії

Цей документ є індексом користувацьких сценаріїв для `mk-combos`. Детальна специфікація кожного сценарію буде додаватися окремо.

## Контексти входу

### Перший запуск

- [`US-001`](./ux/US-001.md): Перший запуск.
  Мета: дати користувачу пройти обов'язкове легке налаштування та перейти до каталогу під час відкриття застосунку без попереднього контексту.
  Статус деталізації: Описано.

### Перший запуск за прямим посиланням

- `US-023`: Перший запуск за прямим посиланням.
  Мета: дати користувачу вперше відкрити конкретну гру, персонажа, combo detail або builder context через URL.
  Статус деталізації: Заплановано.

### Повернення за прямим посиланням

- `US-024`: Повернення за прямим посиланням.
  Мета: дати користувачу з наявними local settings, lists або custom combos повторно відкрити конкретний URL.
  Статус деталізації: Заплановано.

### Внутрішня навігація

Сценарії цього контексту описані в робочих категоріях нижче: каталог, фільтри, detail view, lists, custom combo builder і controller navigation.

## Робочі сценарії

### Початкове налаштування

- `US-008`: Перемикання мови `EN/UA`.
  Мета: дати користувачу змінити мову інтерфейсу та локалізованого контенту.
  Статус деталізації: Заплановано.

- `US-009`: Перемикання відображення інпутів `FGC/PlayStation/Xbox`.
  Мета: показувати одне й те саме combo у зручному для користувача форматі інпутів.
  Статус деталізації: Заплановано.

- `US-021`: Перегляд controller hints.
  Мета: показати користувачу актуальні підказки кнопок для DualSense або Xbox controller.
  Статус деталізації: Заплановано.

### Дослідження каталогу

- `US-002`: Вибір гри `MKXL` або `MK1`.
  Мета: дати користувачу працювати з даними конкретної гри.
  Статус деталізації: Заплановано.

- `US-003`: Вибір персонажа.
  Мета: звузити каталог до комбо конкретного персонажа.
  Статус деталізації: Заплановано.

- `US-004`: Вибір variation для `MKXL`.
  Мета: показати комбо з урахуванням variation персонажа.
  Статус деталізації: Заплановано.

- `US-005`: Вибір kameo для `MK1`.
  Мета: показати комбо з урахуванням kameo-пари.
  Статус деталізації: Заплановано.

- `US-006`: Фільтрація комбо.
  Мета: допомогти користувачу звузити combo list за metadata filters.
  Статус деталізації: Заплановано.

- `US-007`: Перегляд деталей комбо.
  Мета: показати notation, move path, frame timeline, damage, meter, позицію, notes, source і game version.
  Статус деталізації: Заплановано.

### Персональна організація

- `US-010`: Створення named list.
  Мета: дати користувачу створити власний список для тренування або організації комбо.
  Статус деталізації: Заплановано.

- `US-011`: Керування named lists.
  Мета: дати користувачу перейменовувати, видаляти та впорядковувати власні списки.
  Статус деталізації: Заплановано.

- `US-012`: Додавання seeded combo у список.
  Мета: зберегти готове комбо з бази у вибраний користувацький список.
  Статус деталізації: Заплановано.

### Створення власних комбо

- `US-013`: Створення custom combo через guided builder.
  Мета: дати користувачу скласти власне комбо тільки з валідних переходів move graph.
  Статус деталізації: Заплановано.

- `US-014`: Дублювання seeded combo у custom combo.
  Мета: дати користувачу взяти seeded combo за основу для власної версії.
  Статус деталізації: Заплановано.

- `US-015`: Редагування custom combo.
  Мета: дати користувачу змінити власне combo через актуальний guided builder.
  Статус деталізації: Заплановано.

- [`US-025`](./ux/US-025.md): MKXL map interactions у combo builder.
  Мета: дати користувачу будувати або дублювати MKXL combo з урахуванням конкретної карти, zone/segment і environment interactables.
  Статус деталізації: Заплановано.

### Перенесення та відновлення даних

- `US-017`: Експорт full backup.
  Мета: дати користувачу зберегти локальні settings, custom combos і lists у JSON-файл.
  Статус деталізації: Заплановано.

- `US-018`: Імпорт full backup з preview і replace.
  Мета: дати користувачу відновити локальні дані з JSON backup після попереднього перегляду.
  Статус деталізації: Заплановано.

### Керування контролером

- `US-019`: Керування застосунком через DualSense або Xbox controller.
  Мета: дати користувачу керувати catalog, filters, detail view і lists через контролер.
  Статус деталізації: Заплановано.

- `US-020`: Керування combo builder через controller.
  Мета: дати користувачу створювати custom combo в builder через semantic controller commands.
  Статус деталізації: Заплановано.

- `US-022`: Відновлення роботи після disconnect/reconnect controller.
  Мета: забезпечити продовження роботи після втрати або повторного підключення контролера.
  Статус деталізації: Заплановано.

### Стан помилки та відновлення

- `US-016`: Обробка invalid/stale custom combo.
  Мета: показати користувачу, що локальне combo стало невалідним після оновлення move graph, без видалення даних.
  Статус деталізації: Заплановано.

- `US-022`: Відновлення роботи після disconnect/reconnect controller.
  Мета: забезпечити продовження роботи після втрати або повторного підключення контролера.
  Статус деталізації: Заплановано.
